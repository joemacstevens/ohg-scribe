Key Changes

  • Disease Area dropdown replaces inline presets — scales to many categories without horizontal overflow
  • Searchable menu with categories (Oncology, Immunology, Cardiology, etc.) for quick navigation
  • Multi-select — users can load multiple disease vocabularies at once
  • Selected presets as tags — clear visibility of what's active, easy to remove
  • Term count preview — "142 terms" helps users understand what they're adding
  • Conversation type kept — maps to AssemblyAI's speaker_type for smarter identification
  • Speaker names added — optional field to replace A/B/C labels in output
  • "Manage vocabularies" link — future path to settings where admins can add/edit term lists

Implementation Notes Technical

The disease area vocabularies should be stored as JSON files that ship with the app or are fetched from a central config. Structure:

  • vocabularies/oncology/lung-cancer.json → array of boost words
  • When user selects a preset, merge its terms into the boost words input
  • AssemblyAI word_boost param accepts up to 1000 terms for Slam-1, 200 for Universal
  • Consider a "Manage Vocabularies" settings panel where admins can create/edit term lists
